Part 1 · Wrangling

The wrangling workflow

Data wrangling follows six fixed steps: make the data accessible, determine the schema, verify and cast data types, assess missing values, filter to the rows and columns you need, and save your work. "Data cleaning is approximately 80% of data science" — most of the effort in any real analysis is here, before a single statistic gets computed.

The standard stack for nearly every step: numpy for numerical arrays, matplotlib.pyplot for plots, pandas for dataframes, and seaborn for a friendlier statistical plotting API on top of matplotlib. pip install <package> adds any of them to your environment.

Dataframes: variables, observations, values

Loading a dataframe is three lines: df = pd.read_csv('these_data.csv', low_memory=False), then df.shape to verify its dimensions and df.head() to verify its integrity by eye.

Inside that dataframe, columns are variables — features of the phenomenon you are studying — and rows are observations — one instance of that phenomenon. Build a new variable from a list with df[var] = pd.Series(list), or derive one from an existing column with df[var_transform] = transformation(df[var]). Add a new observation by building a one-row dataframe with matching column names and pd.concat([df, new_obs], ignore_index=True). The values are the actual recorded numbers or labels at the intersection of one observation and one variable. "What is an observation?" is one of the most powerful questions in data science — get it wrong and every downstream statistic answers a different question than you think it does.

The schema describes how the data are organized: what tables exist, what variables they contain, what type each variable has, and how tables connect to each other. Data are tabular when they form rows of observations and columns of variables in a rectangular shape. In Pandas, df.columns returns the schema directly.

Variable types and casting

Six conceptually distinct variable types recur constantly: numeric splits into integer (whole numbers, stored as int64) and real-ish (floating point, stored as float64); categorical is a string with a fixed list of labels (color, brand, zip code), usually stored as Pandas' category type; logical is binary True/False; date is a datetime; unstructured text is a string with no fixed label list (reviews, notes); and object is Pandas' catch-all when none of the above quite fits.

To get acquainted with any one variable: its current type (df[var].dtype), how many values are missing (df[var].isna().sum()), what distinct values it takes (df[var].unique()), how often each occurs (df[var].value_counts()), and a histogram of those counts (df[var].hist(bins=35)). From that, you should be able to decide whether a variable needs substantial cleaning before you trust it.

Computers frequently infer the wrong type. A monetary value stored as "-$1,320.15" or a temperature as "23C" gets read as text, because the extra symbols make it look like one. Forcing a variable into a different type than it arrived in is type casting (or coercion): strip the offending symbol (df[var] = df[var].str.replace('$', '')), then convert (df[var] = pd.to_numeric(df[var], errors='coerce')). For other conversions, df[var].astype(str) or .astype(int) handle the rest.

Missing values and imputation

The single most important wrangling problem is missing data — values never recorded for some observations. Mishandling or misunderstanding why data are missing can bias every downstream analysis; missing data is not a mere inconvenience, it is information about how the data were gathered in the first place.

The workflow: count missing values per variable (number_missing = df.isna().sum()), then create a missing-value dummy for every affected column:

schema = df.columns
for var in schema:
    if df[var].isna().any():
        df[f'{var}_NA'] = df[var].isna()

If you must fill the gap, imputation replaces a numeric variable's missing values with its median (df[f'{var}_imp'] = df[var].fillna(df[var].median())) or gives a categorical variable an explicit new category (df[var].fillna('NA') or 'Missing').

Filtering and saving

Column filtering keeps only variables of interest: given a list columns, df[columns] returns just those. Row filtering keeps only observations satisfying a condition: given a boolean row_condition like df[price] > 1000, df[row_condition] returns just those rows. Doing both at once: df.loc[row_condition, columns]. Overwriting df itself with either filter (df = df[columns]) discards the other columns permanently from that variable, forcing a reload if you need them again.

Never overwrite the original data file — always keep it for reproducibility. CSV (df.to_csv('data_cleaned.csv', index=False)) is human-readable and universal; Parquet is more efficient for large or complex data.

Part 2 · Exploratory Data Analysis

A random variable is a phenomenon whose outcome is not yet known; statistics and machine learning guess future values of that phenomenon from past observations. Exploratory data analysis is the step before any of that guessing: discovering the basic features of your variables, one at a time and then in pairs.

Statistics and single variables

Recall the same "getting to know a variable" checklist from wrangling — dtype, missing count, unique values, value counts, histogram — because EDA does not replace wrangling, it depends on it: only a clean-enough variable is worth summarizing.

A statistic is any function that maps data to a number — a proportion (what fraction of observations take a particular label) or a mean (the average of the recorded values) are the two most common. Analyzing a categorical variable \(C\) means computing how often it takes each label: df[var].value_counts(normalize=True) for proportions among non-missing values, adding dropna=False to fold missingness in as its own category, or sns.countplot(data=df, x=var, stat='proportion') to visualize it. Analyzing a numeric variable \(x\) means seeing how its values spread out: sns.histplot(df[var], bins=50) for the shape, df[var].describe() for the standard numeric summary.

Mean and variance

The sample mean is the standard measure of central tendency:

$$M(X) = \frac{x_1+x_2+\cdots+x_n}{n} = \frac{1}{n}\sum_{i=1}^n x_i$$

The sample variance is the average squared distance from that mean, and the sample standard deviation its square root:

$$V(X) = \frac{1}{n}\sum_{i=1}^n \big(x_i - M(X)\big)^2, \qquad \mathrm{sd}(X) = \sqrt{V(X)}$$

Median and quantiles

The mean and variance are particularly sensitive to outliers — atypically high or low values that exert outsized leverage on both. Robust statistics studies estimators less sensitive to them. The sample median is the robust measure of central tendency: the value with half the sample above it and half below.

def median(X):
    values = sorted(X)          # Sort the observations
    n = len(values)              # Number of observations
    middle_index = n // 2        # Determine the middle index

    if middle_index != n / 2:    # If the middle index is odd,
        median = values[middle_index]              # that value is the median
    else:                        # If the middle index is even,
        median = (values[middle_index - 1] + values[middle_index]) / 2  # average values
    return median

The median generalizes to any split point via the \(f\)-th quantile — the value with a fraction \(f\) of the sample below it and \(1-f\) above:

from math import ceil

def quantile(X, frac):
    values = sorted(X)                          # Sort the observations
    n = len(values)                               # Number of observations
    index = max(ceil(n * frac) - 1, 0)            # Find frac-th index
    q = values[index]                             # Determine quantile value
    return q

np.quantile(X, f) does this for you in practice. The robust analog of the variance is the interquartile range, the gap between the 0.75 and 0.25 quantiles:

IQR = np.quantile(X, .75) - np.quantile(X, .50)

The median, IQR, and outliers all show up together in a boxplot — the standard single-picture summary of a numeric variable's robust center and spread. Taming long tails

Almost every real dataset has a handful of variables with long tails — the mean sits far from the median because a small number of very large values pull it away. The traditional fix is the natural logarithm (df['var_ln'] = np.log(df[var])), which compresses large values much more than small ones — but it is undefined at zero and for negative numbers. The better choice is the inverse hyperbolic sine (arcsinh), defined everywhere, including zero and negative inputs, where it evaluates to 0 and to negative outputs respectively:

$$i(x) = \ln\!\left(x + \sqrt{x^2+1}\right)$$

Pairs of variables

Everything so far summarizes one variable. For two categoricals at once, a contingency table (pd.crosstab(df[var1], df[var2])) gives the joint counts, showing how the two vary together or independently.

For two numerics, the visual analog of a contingency table is a scatterplot (sns.scatterplot(x=df[var1], y=df[var2], alpha=.15)), usually summarized numerically by the covariance:

$$\mathrm{cov}(X,Y) = \frac{\sum_{i=1}^n (x_i - M(X))(y_i - M(Y))}{n}$$

computed directly with df[[var1, var2]].cov().

To condition a numeric variable on a categorical one — analyze it separately within each category — split the data by label first. Visually, sns.kdeplot(x=df[num], hue=df[cat], fill=True) overlays one density per group; numerically, df[[cat, num]].groupby(cat).describe() gives a full summary per group, or .mean(), .sum(), .count() for one statistic at a time, or .agg(fcn) for a custom function. When you are hunting for the top or bottom of a group, .sort() at the end orders the result.

FAQ

Why compute missing-value dummies instead of just dropping incomplete rows?

Because whether a value is missing is itself information about the data-gathering process — a survey question skipped disproportionately by one group, a sensor that fails under specific conditions. Dropping incomplete rows silently discards that pattern; keeping a dummy variable lets you check whether "missingness" itself predicts anything, before deciding it's safe to ignore.

Why does imputing with the median matter, specifically, versus the mean?

Because the median is robust to the same outliers that would otherwise distort the mean you'd be filling gaps with. Imputing with a mean that is itself dragged off-center by a few extreme values compounds the original data quality problem instead of routing around it.

Why does covariance divide by n, and not leave the raw sum alone?

So that covariance is an average paired deviation, not a quantity that mechanically grows just because you added more observations — the same reason the sample mean divides by n instead of reporting a raw sum.
