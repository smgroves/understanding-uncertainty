Categorical & Numeric Variables

One indicator function, reused for labels, thresholds, and the ECDF itself
Understanding Uncertainty · lecture notes, digitized

A categorical variable and a numeric variable look like they need entirely different tools — counting labels versus measuring magnitudes. This lecture shows they need the same one. Every question in both halves reduces to "what fraction of my sample satisfies this condition," and every such fraction is an average of a 0/1 indicator, which makes it a Bernoulli trial in disguise — with the same unbiasedness and the same \(p(1-p)/n\) variance every time.

Part 1 · Categorical Variables

Variable and process

Data are samples: realizations of a random variable, drawn from some presumed data generating process. Without that assumption, a learning algorithm has no stable pattern to find — it would just be describing one particular batch of numbers, with no reason to expect the next batch to look similar. The concept connecting a sample back to the process it came from is the random variable and its expectation: how do we expect the process to behave, and what can the data tell us about it?

Categorical variables and the Bernoulli trial

A random variable that generates labels — color, brand, disease type — is categorical. Write it \(C\), with values \(c_i\); car color might be C = ['red', 'blue', 'blue', 'grey', ...], and C[i] one particular observation.

The simplest categorical variable has exactly two outcomes. Flip a coin, call the outcomes \(H\) and \(T\), and assign \(I(H) = 1\), \(I(T) = 0\). \(I\) is an indicator function — it takes the value 1 exactly when a condition (here, "heads") holds. This is a Bernoulli trial: \(p(H) = p\), \(p(T) = 1-p\).

As a full probability space: the sample space is \(\mathcal{S} = \{H, T\}\), the events are \(\mathcal{E} = \{\varnothing, H, T, HT\}\), and the probability function is \(p(\varnothing) = 0\), \(p(H) = p\), \(p(T) = 1-p\), \(p(HT) = 1\). The random variable \(I\) maps \(I(H) = 1\), \(I(T) = 0\).

Expectation, variance, and the unbiased sample proportion

The expectation of the indicator is exactly \(p\):

$$\mathbb{E}[I] = p \times I(H) + (1-p)\times I(T) = p \times 1 + (1-p)\times 0 = p$$

The variance takes a little more algebra, but always resolves to the same shape:

$$\begin{alignat*}{2}
\mathbb{V}[I] &=& \mathbb{E}\big[(I - \mathbb{E}[I])^2\big] \quad &(\text{Definition})\\
&=& p(1-p)^2 + (1-p)(0-p)^2 \quad &(\text{Expand the definition})\\
&=& p(1-p)(1-p+p) \quad &(\text{Factor out } 1-p)\\
&=& p(1-p) \quad &(\text{Simplify})
\end{alignat*}$$

For any 0/1-valued variable, the variance is \(p(1-p)\) — this exact term reappears constantly in this course. Because the trial is 0/1-valued, the sample mean of \(n\) trials is literally the sample proportion of 1s:

$$\hat{p} = \frac{1}{n}\sum_{i=1}^n I_i$$

Linearity of expectation carries the same computation straight through to the sample proportion:

$$\mathbb{E}[\hat{p}] = \mathbb{E}\left[\frac{1}{n}\sum_{i=1}^n I_i\right] = \frac{1}{n}\sum_{i=1}^n \mathbb{E}[I_i] = \frac{1}{n}\sum_{i=1}^n \big(p\times 1 + (1-p)\times 0\big) = p$$

\(\mathbb{E}[\hat{p}] = p\) makes \(\hat{p}\) an unbiased estimator of \(p\): computing a sample proportion is already an estimate of the population probability, with no extra machinery required. The variance of that same sample proportion is \(\mathbb{V}[X]/n\) applied to a Bernoulli — the population variance \(p(1-p)\), divided by \(n\):

$$\mathbb{V}[\hat{p}] = \frac{p(1-p)}{n}$$

Put together: the sample proportion \(\hat{p}\) is an unbiased estimator of \(p\), and its variance shrinks to zero at rate \(1/n\) as the sample grows. Watch for the shape "\(p(1-p)/n\)" — it recurs every time this course asks "what is the sampling variance of a proportion," in this class and later ones.

Multinomial trials and indicator variables

Real categorical variables usually have more than two labels. With \(L\) possible labels \(\ell_1, \dots, \ell_L\) — say four car brands, Honda, Subaru, Ford, Audi — each has its own probability \(p_1, \dots, p_L\), the long-run fraction of the time it occurs.

As a probability space: outcomes are \(\mathcal{S} = \{\ell_1, \dots, \ell_L\}\), and the events are every subset of labels — \(\varnothing\), each single label, every pair, every triple, and so on, up to \(\mathcal{S}\) itself: \(2^L\) events in total. The probability of any tuple of labels is the sum of their individual probabilities.

The indicator function generalizes directly to any one label \(\ell\):

$$\mathbb{I}\{C = \ell\} = \begin{cases} 1, & C = \ell \\ 0, & \text{otherwise} \end{cases}$$

Dummy variables and the sample proportion

Building a full set of indicators — one per label — for a categorical variable is exactly one-hot encoding. In Pandas, pd.get_dummies(df[var], dtype=int) builds all of them; for a model with an intercept, drop one label's column with pd.get_dummies(df[var], dtype=int, drop_first=True) so the columns are not perfectly collinear with the intercept.

The sample proportion of any one label \(\ell\) is the average of its indicator over the sample:

$$\hat{p}_\ell = \frac{1}{N}\sum_{i=1}^N \mathbb{I}\{c_i = \ell\}$$

The exact same argument as the Bernoulli case shows this is unbiased:

$$\mathbb{E}_C[\hat{p}_\ell] = \mathbb{E}_C\left[\frac{1}{n}\sum_{i=1}^n \mathbb{I}\{C_i=\ell\}\right] = \frac{1}{n}\sum_{i=1}^n \mathbb{E}_C\big[\mathbb{I}\{C_i=\ell\}\big] = \frac{1}{n}\sum_{i=1}^n p_\ell = p_\ell$$

You can think of "is \(C\) equal to \(\ell\)" as its own Bernoulli variable: either \(C = \ell\) with probability \(p_\ell\), or \(C \neq \ell\) with probability \(1 - p_\ell\). That Bernoulli's variance is \(p_\ell(1-p_\ell)\), so the sample proportion's variance is the familiar shape again:

$$\mathbb{V}[\hat{p}_\ell] = \frac{p_\ell(1-p_\ell)}{n}$$

Part 2 · Numeric Variables

Numeric variables and why raw plots fail

A numeric variable takes number values — real values like 3.14 (a float) or integers like 25 (an int). The organizing question for this half of the lecture is: how do we analyze a numeric variable the same way we just analyzed a categorical one?

The categorical toolkit does not transfer directly. For a categorical variable, every label typically occurs many times, so counting labels is informative. A numeric variable's exact values are often each observed only once — many values are sparse or unique — so the categorical indicator \(\mathbb{I}\{x_i = x\}\) is nearly useless here: a rug plot of raw values (sns.rugplot()) just shows a scatter of marks with no way to see relative density.

The ECDF

Instead of asking whether \(x_i\) equals a specific value, ask whether it falls at or below a threshold \(x\):

$$\mathbb{I}\{x_i \le x\} = \begin{cases} 1, & x_i \le x \\ 0, & \text{otherwise} \end{cases}$$

Averaging that indicator over the sample gives the proportion of observations below \(x\):

$$\frac{1}{n}\sum_{i=1}^n \mathbb{I}\{x_i \le x\}$$

Fixing this as a function of \(x\) defines the Empirical Cumulative Distribution Function, one of the two main ways this course models a numeric variable:

$$\hat{F}_n(x) = \frac{1}{n}\sum_{i=1}^n \mathbb{I}\{x_i \le x\}$$

The ECDF only changes value where \(x\) sweeps past an observed point \(x_i\), so computing it needs only the unique sorted values of the sample, compared against every point and averaged:

import matplotlib.pyplot as plt

def ecdf(x):
    grid = sorted(x.unique())          # Get the grid
    n = len(x)                          # Sample size
    I = grid.reshape(-1, 1) <= x.reshape(1, -1)  # Compare values
    F_hat = I.sum(axis=0) / n           # Average across columns
    return F_hat, grid

Two structural properties follow directly from the definition: \(\hat{F}_n\) is non-decreasing (moving \(x\) rightward can only capture more observations, never fewer), and it runs from 0 (as \(x \to -\infty\)) to 1 (as \(x \to +\infty\)). At any point \(x\), its value is exactly the proportion of the sample at or below that value.

From ECDF to CDF

The ECDF is an empirical answer to "what fraction of my data is below \(x\)." The population-level version of the same question is the cumulative distribution function, the CDF:

$$F(x) = p[X \le x] = p\big[\{s \text{ in the sample space such that } X(s) \le x\}\big]$$

The connection between the two is that the CDF is exactly the expectation of the same indicator that defines the ECDF:

$$\mathbb{E}\big[\mathbb{I}\{X \le x\}\big] = \sum_x p(x)\,\mathbb{I}\{X \le x\} = F(x)$$

Unbiasedness and variance of the ECDF

Because the ECDF is an average of the indicator \(\mathbb{I}\{X_i \le x\}\), and that indicator has expectation \(F(x)\), the same linearity argument from Part 1 shows the ECDF is unbiased for the CDF at every threshold \(x\):

$$\mathbb{E}_X\left[\frac{1}{n}\sum_{i=1}^n \mathbb{I}\{X_i \le x\}\right] = \frac{1}{n}\sum_{i=1}^n \mathbb{E}_X\big[\mathbb{I}\{X_i \le x\}\big] = \frac{1}{n}\sum_{i=1}^n F(x) = F(x)$$

The observed object, \(\hat{F}_n(x)\), is in expectation exactly the truth, \(F(x)\) — for every threshold \(x\) at once.

Fix one threshold \(x\) and \(\hat{F}_n(x)\) is nothing but a Bernoulli average in disguise: each data point independently lands at or below \(x\) (indicator 1) or above it (indicator 0), with success probability \(p = F(x)\). Every Bernoulli result from Part 1 transfers immediately:

$$\mathbb{V}\big[\hat{F}(x)\big] = \frac{F(x)\big(1 - F(x)\big)}{n}$$

As the sample grows, the ECDF's variance at every threshold shrinks toward zero at rate \(1/n\) — the same guarantee Part 1 proved for a single sample proportion, now holding simultaneously at every point along the whole curve. FAQ

Is a categorical variable with two labels the same thing as a Bernoulli trial?

Yes, exactly — "heads/tails," "disease/no disease," "clicked/did not click" are all Bernoulli trials once you assign 1 to one label and 0 to the other. Every property derived for the coin flip (unbiased sample proportion, variance \(p(1-p)/n\)) applies unchanged to any two-label categorical variable.

Why does one-hot encoding sometimes drop one column?

Because the full set of indicator columns for a categorical variable always sums to exactly 1 in every row — one label indicator is redundant given the rest. If a model already has an intercept term, keeping all \(L\) dummy columns makes that intercept perfectly predictable from the dummies (perfect collinearity), which breaks estimation. Dropping one column removes the redundancy without losing any information: "not any of the remaining labels" already identifies the dropped one.

Why is the ECDF's variance different at different thresholds \(x\)?

Because the variance formula \(F(x)(1-F(x))/n\) depends on \(F(x)\) itself, and \(p(1-p)\) is maximized at \(p = 0.5\) and shrinks toward 0 as \(p\) approaches either 0 or 1. The ECDF is therefore least certain near its own median and most certain out in its tails, where almost all — or almost none — of the data has accumulated.
