Random Numbers & the Sample Mean

Reproducible randomness, and why squared error makes the mean the best guess
Understanding Uncertainty · a build-it-from-scratch walkthrough

Before you can test any statistical idea, you need randomness you can reproduce on demand — the same seed, the same sequence, every time. Once you have that, the first question worth asking is: given a sample of patients, what single number best predicts the next one? Calculus has an answer, and it is not a guess — it is exactly the sample mean, the unique number that minimizes squared error against the data you already have.

Built on the Class 04 notebooks 04_1_learning_from_data and 04_2_random_number_generation — see the full lecture for every derivation this lab only sketches. Data: 40 patients' ages (CardiacPatientData.csv, uu_fa26/class_02/).

The best guess: minimizing squared error

Each term in the sum being minimized is \((\text{age}_i - \hat{x})^2\) — a single parabola in \(\hat{x}\), shifted to be centered at \(\text{age}_i\). Summing quadratics with the same leading coefficient always gives back a quadratic — the shape can shift left or right, but it can never grow a second dip.
  • Share · 2 minAgree on what that guarantees about how many minima the curve can have, then check the derivation below.

Predicting a new patient's age with a single guess \(\hat{x}\), the natural way to score yourself is mean squared error against the sample you have:

$$\text{MSE}(\hat{x}) = \frac{1}{n}\sum_{i=1}^n (x_i - \hat{x})^2$$

Because that curve is always a parabola opening upward, it has exactly one minimum, found by setting its derivative to zero:

$$\frac{d}{d\hat{x}}\,\text{MSE}(\hat{x}) = \frac{1}{n}\sum_{i=1}^n 2(x_i - \hat{x})\cdot(-1) = 0 \quad\Longrightarrow\quad \hat{x} = \frac{1}{n}\sum_{i=1}^n x_i$$

The minimizing guess is the sample mean — not by convention, but because it is the unique answer to "which single number is closest, in squared distance, to all of these numbers at once." The full derivation of why that same mean is also an unbiased estimator with variance \(\mathbb{V}[X]/n\) is in the lecture; this section is just the "why the mean, and not the median or something else" half of the story.

NumPy's rng.choice(A, size=N) draws with replacement by default: every draw sees the full set again, so the same element can appear more than once. Passing replace=False removes each drawn element from further consideration, so \(N\) draws from an \(N\)-element set is always a full permutation.

Assignment

Implement the recurrence behind a seeded random number generator, and a sampling function built on top of it — no built-in random module, no shortcuts. You are given the data loader and the input/output plumbing the autograder speaks. You write two functions.

What you build

  • lcg_next(state) — advance a linear congruential generator one step: given the current integer state, return (new_state, uniform) where uniform is a float in [0, 1).
  • choice(items, k, seed, replace) — draw k elements from the list items, using repeated calls to lcg_next seeded with seed, with or without replacement depending on the flag.

I/O protocol

The grader launches python3 rng.py and sends one query per line on stdin:

next <state>                          -> OK: <new_state> <uniform, 6 decimals>
choice <items,comma,sep> <k> <seed> <0|1>   -> OK: <k comma-separated items>
ERR: unknown query                      # anything else, or k too large without replacement

The last argument to choice is 1 for with-replacement, 0 for without.

FAQ

Why does the MSE curve have to be a parabola?

Because it is a sum of terms of the form \((x_i - \hat{x})^2\), and expanding the square shows every term is quadratic in \(\hat{x}\) with the same leading coefficient (1, after expansion). Summing quadratics that share a leading coefficient always produces another quadratic — the sum can never develop a second bend.

Is the sample mean always the best guess?

Only under squared error. Switch the loss function to mean absolute error, \(\frac{1}{n}\sum |x_i - \hat{x}|\), and the minimizing guess becomes the median instead. The "best" predictor is a property of the loss function you chose, not an absolute fact about the data.

Why does a real linear congruential generator need such a specific-looking formula?

The multiplier, increment, and modulus in a real LCG are chosen so the generator has the longest possible cycle before it repeats itself and so consecutive outputs show no easily detectable correlation. A carelessly chosen set of constants can produce a generator that repeats after only a few thousand draws, or whose low-order bits cycle in an obvious pattern — both of which numpy's own generator is engineered to avoid.

Does vectorized code always beat a loop?

Almost always for array-at-a-time numeric work, but not by magic — it wins because the per-element work happens inside pre-compiled, optimized code instead of the interpreter's own step-by-step execution. If the "vectorized" version has to allocate a much larger intermediate array than the loop ever holds in memory, that memory cost can erase some or all of the advantage. Measure on your own machine before assuming.
